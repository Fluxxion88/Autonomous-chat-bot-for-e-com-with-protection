# Data Model: Proof of Audit

## 1. DB1_Actions
This table logs the raw text outputs from the AI agent for the dashboard display.

| Field | Type | Attributes | Description |
|---|---|---|---|
| `id` | UUID/String | Primary Key | Unique identifier for the transaction. Shared with DB2. |
| `timestamp` | Timestamp | UTC ISO 8601 | When the transaction occurred. |
| `discount_val`| String | Not Null | The numeric string representation of the discount generated by the agent (e.g., "20", "100"). Must be `.isdigit()`. |

## 2. DB2_AuditLog
This table logs the cryptographic hashes of the decisions to detect tampering.

| Field | Type | Attributes | Description |
|---|---|---|---|
| `id` | UUID/String | Primary Key | Unique identifier for the transaction. Shared with DB1. |
| `timestamp` | Timestamp | UTC ISO 8601 | When the transaction occurred. |
| `discount_hash`| String | Not Null | The SHA-256 hash of the UTF-8 encoded `discount_val` from DB1. |

### Validation Rules
- Both tables are updated in a single transaction.
- If an insert into `DB2_AuditLog` fails, the `DB1_Actions` insert MUST be rolled back.
- `DB2_AuditLog` is APPEND-ONLY. No `UPDATE` or `DELETE` allowed via UI/API.
- `discount_val` MUST be validated as `val.isdigit() == True` before insertion.
